Tips and tricks I learned redesigning my website

I have spent the past few days redesigning my website from the ground up. I kept running into problems when I wanted to add new features to my website and the color scheme made things very difficult to read. Today, I will share some of the things I learned when redesigning my site. First, take a look at the old layout. While the background picture is awesome, it created too many color challenges and I had to start over.

Using HTML5 elements to organize sections

I wanted to create a fixed header at the top of my blog with navigation links to the rest of my site. There are many ways to do this, and I chose to use the (header) element to seperate everything. Below is the CSS portion of my header.

header {
width: 100%;
position: fixed;
background: black;
z-index: 10000000;
top: 0;
height: 65px;}

This is one of the simpler ways to create a header, but I found it very effective. The "position:fixed;" command creates the outline for our header while the width, height, and top commands create its dimensions. The z-index ensures that no other elements will scroll on top of our header, the higher the number the higher priority it has over other elements. I chose a very high number so any element with a z-index lower that this will be shown under our header.

Using screen percentages to add my header elements

Again, there are many ways to add elements to our header, but I chose to wrap each element in its own "div" and position it according to its percentage of the width of the screen. Try resizing this browser and notice how the links in the top right corner move as the browser is resized. I thought this was a great way to position my fixed elements. The images do not overlap unless the window is too small to be useful. These elements also look great on smaller screens such as my smartphone and tablet. Here is my CSS for my google plus link, as well as the link to send me money through paypal. Try the paypal link for yourself, donations are greatly appreciated!

#googleicon{
position:fixed;
top:10px;
left:82%;}

#paypalicon{
position:fixed;
top:10px;
left:88%;}

Creating a navigation bar using lists

At its core, a navigation bar is simply an unordered list. The way it looks depends on the styles you choose in css. In order to blend our list into our header we simply gave it the same background color. To create this navigation bar, we simply created an unordered list in HTML and wrapped our list in a div called "nav." Here is the CSS for the structure of our navigation bar:

#nav{
width: 100%;
float: left;
background-color: black;}

#nav ul{
list-style-type: none;
width: 800px;
margin: 0 auto;}

#nav li{
float: left;}

#nav a:link, a:visited {
display: block;
width: 230px;
text-decoration: none;}

In our first call to our div, we created a width that extends the whole screen and gave it the same background color as our header so it looks like its included header, however when you scroll down the screen the navigation bar disappears to make room for the blog. I next removed the styling from the lists within that div so we could set the dimensions of our bar. I next floated lits to the left so the links are displayed side-by-side. Finally we styled our links and applied a standard width to each link.
