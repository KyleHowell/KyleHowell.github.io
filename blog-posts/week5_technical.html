Organizing data into classes using Ruby



Ruby is great at organizing information. One way to group together items that share similar characteristics is with Classes. Lets say we wanted to make a database of restaurants in San Francisco. We could do this a number of ways, lets first try using a Hash.

Restaurants = {"Millenium" => {type: "Vegetarian",average_check: 60,location: "Tenderloin"},
"The Waterfront" => {type: "American",average_check: 70,location: "North Beach"},
"Fleur de Lys" => {type: "French",average_check: 100,location: "Nob Hill"}}

Notice all of our Restuarants share the same characteristics. All of them have a location, average_check, and type of food they serve. Since all of these items share the same characteristics, this is a great opportunity to use classes. This will help organize our information and access information easier. If we wanted to make a change to every Restaurant, we could simply make changes to our class and every instance of that class would change. Lets try organizing the same information using Classes. We must first make a "blueprint" for the characteristics of new members of the class. This is done using the initialize method. Initialize stores information in instance variables that we can use when writing methods for our Class. Consider this example:

class Restaurant
def initialize(name, type, avg_check, location)
@name = name
@type = type
@avg_check = avg_check
@location = location
end
end
Millenium = Restaurant.new("Millenium", "Vegetarian", 60, "Tenderloin")
Waterfront = Restaurant.new("The Waterfront", "American", 70, "North Beach")
Fleur_de_Lys = Restaurant.new("Fleur De Lys", "French", 100, "Tenderloin")

Now we have created objects of the class Restaurant that share the same input. However, we cannot access the information in our new objects without first creating methods that access them. Even though the information is stored as an instance variable, we must write a method that gives us access to the instance varable. Or we could use 'attr_reader', which is the same as adding the code below.

def type
return @type
end
p Millenium.type
OUTPUT: "Vegetarian"

Whenever we make a call to the type method it will return a String containing the type of Restaurant that it is. We just add the folowing code to the top of our class.

attr_reader :type

Now lets add a method that prints out a summary of the characteristics of a restaurant. This can be done with just a few lines of code since all of our informaion is stored in the instance variables of each object. Consider the following code:

def summary
puts "#{@name} is a #{@type} restaurant in the #{@location} district. Its average check is #{@avg_check} dollars per person."
end
Millenium.summary
OUTPUT: "Millenium is a Vegetarian restaurant in the Tenderloin district. Its average check is 60 dollars per person."

Overall, Classes are a great way to organize objects that share similar characteristics. Here is our completed Restuarant Class with proper syntax. As well as an example of how to print our a summary of each Restaurant that we have.

class Restaurant
def initialize(name, type, avg_check, location)
@name = name
@type = type
@avg_check = avg_check
@location = location
end
def type
return @type
end
def summary
puts "#{@name} is a #{@type} restaurant in the #{@location} district. Its average check is #{@avg_check} dollars per person."
end
end

Millenium = Restaurant.new("Millenium", "Vegetarian", 60, "Tenderloin")
Waterfront = Restaurant.new("Waterfront", "American", 70, "North Beach")
Fleur_de_Lys = Restaurant.new("Fleur de Lys", "French", 100, "Tenderloin")
Millenium.summary
Waterfront.summary
Fleur_de_Lys.summary

OUTPUT:
Millenium is a Vegetarian restaurant in the Tenderloin district. Its average check is 60 dollars per person.
Waterfront is a American restaurant in the North Beach district. Its average check is 70 dollars per person.
Fleur de Lys is a French restaurant in the Tenderloin district. Its average check is 100 dollars per person.
