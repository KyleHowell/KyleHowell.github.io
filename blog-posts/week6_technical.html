Ruby Classes and Modules

Working with Ruby for a couple of weeks now, we have become familiar working with classes. Classes are great because if objects all share the same characteristics, then organizing them in a class makes sense. But what if you have multiple classes that all want to use the same method? This is where modules come in. Modules are about providing methods that you can use across multiple classes. In cases where you want to group code that belongs together, but you do not need to create an object from that code, modules are a great way to organize that information. Lets say you wanted to create a class that initializes a single variable called name. You may have mulitple classes that initialize the same variable, so lets use a module to represent that information.

module test
def initialize(name)
@name = name
end
def return_name
@name
end
end

class Class1
include test
end
class Class2
include test
end

Kyle = Class1.new("Kyle")
Steve = Class2.new("Steve")
p Kyle.return_name
p Steve.return_name
OUTPUT:
"Kyle"
"Steve"

In this example, we used a module to represent the initialize method as well as our return_name method for both of our classes. By using the include keyword followed by the module name, we included our module in our class. As you can see, modules are a great way of reusing information multiple times. Since both of our classes initialize the same variable and share the same method, we only had to write out these methods once!

Ruby has built in modules that are very useful. The Math module allows us to access constants such as PI and E without having to define these number ourselves! It also contains methods such as sine and log that we can use by accessing this module.
